Objetivo: Criar um aplicativo web de conferência de carga com três módulos: Administrador, Equipe de entrega e Supervisor de Rota. O sistema deverá ter a logo da empresa e layout nas cores mostarda e azul (cores da logo). O nome do app pode ser ConfereJá. 
Módulo Administrador deverá ter Dashboard com indicadores: tempo de conferência de cada mapa, tempo médio de conferência da sala Corona, tempo médio de conferência da sala Stella, % de divergência encontrada em cada mapa, % de avarias relatada em cada mapa, % de contagens parciais. Pasta de Upload de relatórios (Excel e csv). Deverá ter uma pasta para “base de motoristas”, uma pasta para relatório “031120 PW” e uma pasta para relatório “Detalhe Separação WMS”. O relatório do WMS irá conter informações de mapa, baias, produtos, quantidades e unidades de medida. Controle de usuários e permissões. Visualização de conferências realizadas, incluindo tempo total e contagens parciais. Exportação de relatórios em PDF/Excel. Módulo Equipe de Entrega Login: Campos: Matrícula do motorista, mapa do caminhão. Após login, exibir as baias do caminhão conforme configuração: Lado do ajudante: A01, A02, A03 Lado do motorista: M01, M02, M03. Fluxo de conferência: Ao clicar em uma baia, listar todos os produtos (SKU), descrição, quantidade e unidade de medida. Ordem de montagem conforme relatório de detalhe separação WMS enviado pelo administrador. Relatório Excel: Coluna B: número do mapa. Coluna C: baia do caminhão (ex.: P03_A_03_1/42 → baia A03; P01_M_01_1/42 → baia M01). A letra A indica lado do ajudante. A letra M indica lado do motorista. O número após a letra indica o número da baia. Coluna G: código do produto. Coluna H: descrição do produto. Coluna I: quantidade. Coluna L: unidade de medida. Cada item terá: Checkbox para marcar como conferido. Campo opcional para contagem parcial (quando não for possível contar 100%). Controle de tempo O sistema deverá registrar o tempo total de conferência: Início: quando o motorista abre a primeira baia. Fim: quando todas as baias forem conferidas ou quando o motorista finalizar manualmente. O tempo deverá ser armazenado e exibido no dashboard do administrador. Requisitos técnicos Interface responsiva (desktop e mobile). Autenticação segura (JWT ou OAuth2). Banco de dados relacional para garantir integridade dos dados. Arquitetura modular para facilitar manutenção. Boas práticas de UX/UI com cores mostarda e azul. Upload e leitura de arquivos Excel para alimentar dados das baias. Sugestão de stack Frontend: Next.js + Tailwind CSS + TypeScript. Backend: Node.js + NestJS + TypeScript. Banco de dados: PostgreSQL. Infraestrutura: Docker + AWS (EC2, RDS, S3 para documentos). Autenticação: Auth0 ou Firebase Auth. Upload e leitura de Excel: xlsx ou SheetJS. Mapas: Leaflet.js ou Mapbox. Gráficos: Chart.js ou Recharts. Upload e processamento de relatórios Excel. Interface para conferência por baia com checkbox e campo de contagem parcial. Registro e cálculo do tempo de conferência. Dashboard com métricas e relatórios.
